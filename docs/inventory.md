# Inventory — blocker & decision log

**Convention:** blockers get logged here and work continues. The run never pauses to
ask. When something blocks, write the entry, take the documented fallback, keep
going. Unresolved entries are triaged in Phase 6, not mid-flight.

Statuses: `open` · `fallback-taken` · `resolved` · `accepted` (we're living with it).

## How to add an entry

Append to the table. Keep the fallback column filled in — an entry without a
fallback is an unanswered question, which is the one thing that can stall a run.

# Inventory ‚Äî blocker & decision log

**Convention:** blockers get logged here and work continues. The run never pauses to
ask. When something blocks, write the entry, take the documented fallback, keep
going. Unresolved entries are triaged in Phase 6, not mid-flight.

**Anti-race ‚Äî NEVER `git stash` on the shared tree.** Several roles hold uncommitted work
here at once. `git stash` is tree-wide: it snatches *other* roles' in-flight edits, and a
`pop` that conflicts leaves someone else's work in a state they did not create and cannot
see coming. The rule: **commit your own pathspec first, then sync.** `git commit -- <your
paths>` then `git pull --ff-only` touches nothing but yours. If you must sync before you
can commit, the answer is to commit a smaller pathspec, not to stash. (Learned during D21:
syncing inventory.md at HEAD required stashing the scout's mid-flight `game/` tree. It
round-tripped cleanly and nothing was lost ‚Äî that was luck, not design.)

Statuses: `open` ¬∑ `fallback-taken` ¬∑ `resolved` ¬∑ `accepted` (we're living with it).

**ID allocation ‚Äî pull and re-check HEAD immediately before push; on collision the later writer moves.**
Claim the next free number from HEAD, not from a ledger or a stale read. A number is only real once it
is committed in this file, and concurrent writers *will* race the same number otherwise (B23‚ÄìB26 cost
~3 turns; B41 raced again). If the later writer's ID is already cited elsewhere in the repo, move
whichever side has fewer references instead ‚Äî the goal is a clean map, not a moral one.

**Standing convention ‚Äî every boot/CI check must be negative-controlled.** Deliberately break the
condition and confirm the check *fails* before trusting that it passes. A check that has never been
seen to fail is not evidence; it is decoration. Earned twice: **B18** (`SIM_CHECK` validated the tick
rate against its own constant, so it returned PASS at 20 Hz while the scaffold contradicted the spec
‚Äî it could not have caught the bug it appeared to cover) and again in **B22**'s `filter_check`. This
is the concrete form of **B34**'s meta-finding: reasoned-but-unmeasured claims are the recurring bug
class here, and a green check nobody has falsified is exactly that class wearing a test's clothing.

## How to add an entry

Append to the table. Keep the fallback column filled in ‚Äî an entry without a
fallback is an unanswered question, which is the one thing that can stall a run.
